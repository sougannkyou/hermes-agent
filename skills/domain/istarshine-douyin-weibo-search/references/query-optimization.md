## 扩展规则

1. **保留原始短语**：原始关键词作为第一个搜索请求
2. **拆解为核心实体 + 事件词组合**：将关键词拆解为核心实体（人名、国名、组织等）和事件动词/名词，用空格（AND）组合
3. **补充同义词和相关表述**：添加同一事件的常见替代说法
4. **补充关键人物/地点**：添加与该事件强相关的专有名词

查询使用 Google 搜索运算符语法（`q` 参数），多个关键词空格分隔表示 AND 关系。由于 `q` 参数不支持 OR 语法，需要对每个扩展分支分别发起搜索请求，或选择覆盖面最广的一个分支。

## 示例

### 示例 1：美伊战争

用户输入：`美伊战争`

扩展后的查询（按优先级选择，或分别搜索）：

```
# 分支 1：原始短语
q=美伊战争

# 分支 2：同义词
q=美伊冲突

# 分支 3：核心实体 + 事件词
q=美国 伊朗 袭击

# 分支 4：关键地点
q=霍尔木兹海峡
```

实际使用时，优先用分支 1 搜索，如果结果不足再补充其他分支。

### 示例 2：新能源汽车

用户输入：`新能源汽车`

扩展后的查询：

```
# 分支 1：原始短语
q=新能源汽车

# 分支 2-3：同义词
q=新能源车
q=电动汽车

# 分支 4：核心实体 + 事件词
q=新能源 销量
```

### 示例 3：结合运算符的完整查询

```
# 搜索抖音平台最近 7 天的大V发布的新能源汽车内容
q=新能源汽车 site:iesdouyin.com dateRestrict:d7 followers:500000..

# 搜索高互动的小米汽车评论，按点赞数排序
q=小米汽车 likes:10000.. dateRestrict:d7

# 搜索特定作者的正面情感帖子
q=新能源 author:张三 sentiment:positive
```

### 可用运算符速查

| 运算符 | 说明 | 示例 |
|--------|------|------|
| `site:` | 平台过滤 | `site:iesdouyin.com` |
| `dateRestrict:` | 时间范围 | `dateRestrict:h1`、`dateRestrict:d7` |
| `author:` | 作者匹配 | `author:张三` |
| `sentiment:` | 情感过滤 | `sentiment:positive` |
| `followers:` | 粉丝数筛选 | `followers:500000..` |
| `replies:` | 评论数筛选 | `replies:10000..` |
| `likes:` | 点赞数筛选 | `likes:5000..` |
| `reposts:` | 转发数筛选 | `reposts:1000..` |
| `-` 前缀 | 排除条件 | `-site:weibo.com` |

完整运算符列表见 SKILL.md 的「Google 搜索运算符语法」章节。

## 迭代优化

首次搜索后，根据 stats 返回的 hotWords 和 hotHashtags 结果，可进一步补充扩展词：

1. 从 `hotWords` 结果中提取高频相关词，作为新的搜索分支
2. 从 `hotHashtags` 结果中提取相关话题标签，作为新的关键词搜索
3. 如果首次搜索结果量偏少（< 50），应放宽条件重新搜索（减少关键词、扩大时间范围）

## 注意事项

- 扩展查询中的运算符（`site:`、`dateRestrict:` 等）在每个分支中保持一致
- 不要过度扩展导致噪音过多，一般 3-6 个分支即可
- 对于已经很具体的短语（如人名"拉里贾尼"），无需扩展，直接使用
- 范围筛选语法：`min..max`（双边）、`min..`（仅下限）、`..max`（仅上限）、`数值`（等价于 ≥ 该值）
