## 扩展规则

1. **保留原始短语**：原始关键词作为第一个搜索请求
2. **拆解为核心实体 + 事件词组合**：将关键词拆解为核心实体（人名、国名、组织等）和事件动词/名词，用空格（AND）组合
3. **补充同义词和相关表述**：添加同一事件的常见替代说法
4. **补充关键人物/地点**：添加与该事件强相关的专有名词
5. **多语言翻译（境外全网搜必须）**：如果用户输入的是中文关键词，必须将核心关键词翻译为主要语种作为额外搜索分支。境外数据以外语为主，仅用中文搜索会遗漏绝大部分内容。建议翻译的语种：
   - 英语（en）— 覆盖面最广，必须翻译
   - 法语（fr）
   - 西班牙语（es）
   - 葡萄牙语（pt）
   - 俄语（ru）
   - 阿拉伯语（ar）
   - 日语（ja）
   - 韩语（ko）
   
   根据话题相关性选择 3-5 个语种即可，不必全部翻译。英语分支优先级最高。

查询使用 Google 搜索运算符语法（`q` 参数），多个关键词空格分隔表示 AND 关系。由于 `q` 参数不支持 OR 语法，需要对每个扩展分支分别发起搜索请求，或选择覆盖面最广的一个分支。

## 示例

### 示例 1：美伊战争

用户输入：`美伊战争`

扩展后的查询（按优先级选择，或分别搜索）：

```
# 分支 1：原始短语
q=美伊战争

# 分支 2：同义词
q=美伊冲突

# 分支 3：核心实体 + 事件词
q=美国 伊朗 袭击

# 分支 4：关键地点
q=霍尔木兹海峡
```

实际使用时，优先用分支 1 搜索，如果结果不足再补充其他分支。

### 示例 2：新能源汽车

用户输入：`新能源汽车`

扩展后的查询：

```
# 分支 1：原始中文短语
q=新能源汽车

# 分支 2-3：中文同义词
q=新能源车
q=电动汽车

# 分支 4：英语翻译（必须）
q=new energy vehicle
q=electric vehicle

# 分支 5：法语翻译
q=véhicule électrique

# 分支 6：西班牙语翻译
q=vehículo eléctrico

# 分支 7：俄语翻译
q=электромобиль
```

实际使用时，优先搜索英语分支（覆盖面最广），再根据需要补充其他语种。

### 示例 3：多语言翻译 — 人工智能

用户输入：`人工智能`

```
# 分支 1：中文原始
q=人工智能

# 分支 2：英语（必须）
q=artificial intelligence

# 分支 3：英语缩写
q=AI

# 分支 4：法语
q=intelligence artificielle

# 分支 5：西班牙语
q=inteligencia artificial

# 分支 6：日语
q=人工知能

# 分支 7：韩语
q=인공지능

# 分支 8：阿拉伯语
q=الذكاء الاصطناعي
```

根据话题的地域相关性选择语种。全球性话题优先英语+法语+西班牙语；亚洲话题加上日语+韩语；中东话题加上阿拉伯语。

### 示例 4：结合运算符的完整查询

```
# 搜索所有平台最近 7 天的英语 AI 相关内容
q=AI lang:en dateRestrict:d7

# 搜索最近 7 天的日语新能源汽车内容
q=新能源汽车 lang:ja dateRestrict:d7

# 搜索高互动的英语小米汽车评论，按点赞数排序
q=小米汽车 lang:en likes:10000.. dateRestrict:d7

# 搜索特定作者的正面情感帖子，排除阿拉伯语
q=新能源 author:张三 sentiment:positive -lang:ar

# 搜索所有平台最近 7 天的大V发布的韩语内容
q=新能源汽车 lang:ko dateRestrict:d7 followers:500000..
```

### 可用运算符速查

| 运算符 | 说明 | 示例 |
|--------|------|------|
| `site:` | 平台过滤（支持所有平台域名） | `site:twitter.com` |
| `dateRestrict:` | 时间范围 | `dateRestrict:h1`、`dateRestrict:d7` |
| `author:` | 作者匹配 | `author:张三` |
| `sentiment:` | 情感过滤 | `sentiment:positive` |
| `lang:` | 语言过滤 | `lang:en`、`lang:ja`、`-lang:ar` |
| `followers:` | 粉丝数筛选 | `followers:500000..` |
| `replies:` | 评论数筛选 | `replies:10000..` |
| `likes:` | 点赞数筛选 | `likes:5000..` |
| `reposts:` | 转发数筛选 | `reposts:1000..` |
| `-` 前缀 | 排除条件 | `-site:weibo.com`、`-lang:ar` |

完整运算符列表见 SKILL.md 的「运算符语法与字段」章节。

## 迭代优化

首次搜索后，根据 stats 返回的 hotWords 和 hotHashtags 结果，可进一步补充扩展词：

1. 从 `hotWords` 结果中提取高频相关词，作为新的搜索分支
2. 从 `hotHashtags` 结果中提取相关话题标签，作为新的关键词搜索
3. 如果首次搜索结果量偏少（< 50），应放宽条件重新搜索（减少关键词、扩大时间范围）
4. 利用 `langDistribution` 统计结果了解各语言的数据分布，针对数据量大的语言进行深入分析

## 注意事项

- **中文关键词必须翻译**：境外数据以外语为主，仅用中文搜索会遗漏绝大部分相关内容。英语翻译是必须的，其他语种根据话题选择
- 扩展查询中的运算符（`site:`、`dateRestrict:`、`lang:` 等）在每个分支中保持一致
- 不要过度扩展导致噪音过多，中文分支 3-6 个 + 外语分支 3-5 个语种即可
- 对于已经很具体的短语（如人名"拉里贾尼"），无需扩展同义词，但仍需翻译为外语（如 Larijani）
- 范围筛选语法：`min..max`（双边）、`min..`（仅下限）、`..max`（仅上限）、`数值`（等价于 ≥ 该值）
- 境外全网搜覆盖境外社交媒体平台的多语言数据，不使用 `site:` 运算符时默认搜索所有平台
- 使用 `lang:` 运算符可以精准过滤特定语言的数据，使用 `-lang:` 可以排除特定语言
